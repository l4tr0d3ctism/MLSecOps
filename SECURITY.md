# Security Policy

## Reporting a Vulnerability

If you discover a security issue in this repository (for example, exposed credentials, malicious content in a chapter, or a vulnerability in the documentation build pipeline), please **do not** open a public GitHub issue with exploit details.

Instead:

1. Open a [GitHub Security Advisory](https://github.com/l4tr0d3ctism/MLSecOps/security/advisories/new) (preferred), or
2. Open a private issue describing the impact and reproduction steps without public exploit code.

We will acknowledge reports within a reasonable timeframe and coordinate disclosure.

## Scope

This policy covers the **MLSecOps Guide repository** (Markdown content, MkDocs site, GitHub Actions workflows). It does **not** cover third-party tools, vendors, or AI systems described in the guide.
